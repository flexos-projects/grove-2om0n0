# Database & Data Model — Grove

## Stack Note

**Database: Convex** — chosen for real-time sync (essential for a messaging product), built-in auth, and developer-friendly schema definitions. The inbox must update instantly when a guest texts — polling is not acceptable. Convex's reactive queries solve this natively.

**File Storage: Convex file storage** — for property logos and image attachments in conversations. Capped at 10MB per file.

## Denormalised Fields

| Field | Location | Source | Update Trigger |
|-------|----------|--------|----------------|
| `conversationCount` | `properties` | Count of `conversations` this month | On conversation creation, reset on 1st of month |
| `totalConversations` | `guests` | Count of `conversations` for this guest | On conversation creation |
| `lastContactAt` | `guests` | Latest `conversation.lastMessageAt` | On message creation |
| `lastMessageAt` | `conversations` | Latest `message.createdAt` | On message creation |
| `lastMessagePreview` | `conversations` | Truncated `message.body` | On message creation |
| `messageCount` | `conversations` | Count of `messages` | On message creation |

## Tier Enforcement

| Limit | Starter | Professional | Group |
|-------|---------|-------------|-------|
| Conversations/month | 500 | Unlimited | Unlimited |
| Staff seats | 2 | 5 | 15/property |
| Knowledge entries | 50 | Unlimited | Unlimited |
| Properties | 1 | 1 | 3-20 |

Enforcement is at the server/mutation layer:
- **Conversation limit:** Check `properties.conversationCount` before creating a new conversation. If at limit, AI auto-response stops; staff can still respond manually.
- **Seat limit:** Check `propertyMembers` count before creating an invite.
- **Knowledge limit:** Check `knowledgeEntries` count before creating an entry.

## File Storage

- **Property logos:** Uploaded during onboarding or settings. Max 5MB, PNG/JPG/SVG. Stored via Convex file storage, URL saved to `properties.logoUrl`.
- **Message attachments:** Staff can attach images to outbound messages. Max 10MB, PNG/JPG. Stored via Convex, URL saved to `messages.mediaUrl`. Inbound MMS media URLs are Twilio-hosted and stored directly.
