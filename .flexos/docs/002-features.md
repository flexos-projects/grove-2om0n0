# Feature Inventory — Grove

## Plan Tiers

### Starter — $149/month per property

For single-property hotels testing the waters.

- 1 property
- Up to 500 SMS conversations/month
- AI auto-responses (standard templates)
- 2 staff seats
- Email support
- Grove branding on SMS footer ("Powered by Grove")

### Professional — $349/month per property

For properties ready to make Grove their concierge standard.

- 1 property
- Unlimited SMS conversations
- AI auto-responses with custom voice training
- 5 staff seats
- Request routing and assignment
- Guest preference memory
- White-label SMS (no Grove branding)
- Priority support

### Group — $249/month per property (minimum 3)

For boutique hotel groups running multiple properties.

- 3-20 properties under one account
- Everything in Professional
- 15 staff seats per property
- Cross-property guest recognition
- Group-level analytics dashboard
- Dedicated onboarding specialist
- API access for PMS integration

## Feature Area: Authentication & Account Setup

### F-001: Hotel Onboarding

Staff member creates an account, names their property, and configures basic details (property name, address, timezone, SMS phone number). Guided flow — 4 steps, under 5 minutes. The phone number is provisioned automatically via Twilio during onboarding.

**Key behaviours:**
- Email/password signup (no social login — this is a business tool)
- Property name, address, timezone, and description
- SMS number selection (area code preference, auto-provisioned)
- Upload property logo (used in dashboard, not guest-facing)
- Invite first team member (optional, skippable)

**Tier:** All tiers.

### F-002: Team Management

Property owner invites staff members by email. Two roles: Admin (full access, billing, settings) and Staff (can view and respond to conversations, no billing or settings access). Admins can remove team members.

**Key behaviours:**
- Invite by email → recipient creates password
- Role assignment: Admin or Staff
- Remove team member (admin only, requires confirmation)
- Seat limits enforced per plan tier
- Last admin cannot be removed

**Tier:** All tiers (seat limits differ).

### F-003: Login / Logout

Email and password authentication. Persistent session (30 days). No magic links, no SSO — keep it simple for v1.

**Key behaviours:**
- Email + password login
- "Forgot password" → reset email
- Session persists 30 days unless explicitly logged out
- Redirect to dashboard after login

**Tier:** All tiers.

## Feature Area: SMS & Conversations (Core Product Loop)

### F-004: Inbound SMS Reception

Guest texts the hotel's Grove number. Message appears in the dashboard conversation view in real time. If no conversation exists for that phone number, a new one is created. If a conversation exists, the message is appended.

**Key behaviours:**
- Messages arrive via Twilio webhook → stored in database → pushed to dashboard via real-time sync
- New phone number → new conversation created with "Unknown Guest" label
- Returning phone number → message appended to existing conversation
- Media messages (photos) supported — displayed inline
- Messages timestamped in hotel's local timezone

**Tier:** All tiers.

### F-005: AI Auto-Response

When a message arrives, the AI evaluates it and either: (a) sends an instant response for simple requests, or (b) flags it for human review. The AI uses the hotel's property info, local knowledge base, and any custom voice training to generate responses.

**Key behaviours:**
- AI classification: auto-respond vs. needs-human (confidence threshold configurable)
- Auto-responses sent within 10 seconds
- Auto-responses logged with "AI" badge in conversation view
- Staff can review and edit AI responses retroactively
- AI tone matches the hotel's configured voice (casual, formal, or warm — default: warm)
- Categories auto-handled: directions, hours, local recommendations, hotel amenities info, weather, basic FAQs
- Categories flagged for human: complaints, complex bookings, emotional messages, anything mentioning safety or health, requests the AI isn't confident about

**Tier:** All tiers (Starter gets standard templates, Professional+ gets custom voice training).

### F-006: Staff Response

Staff member reads a guest message and types a response. The response is sent as SMS from the hotel's Grove number. The guest sees it as a normal text message from the same number they texted.

**Key behaviours:**
- Text input with send button
- AI draft suggestion shown (staff can use, edit, or ignore)
- Response sent via Twilio
- Sent messages timestamped and attributed to the staff member who sent them
- Staff can attach images (e.g., a photo of a menu or map)
- Character count indicator (SMS segment awareness)

**Tier:** All tiers.

### F-007: Conversation Management

Staff can view all conversations in a list, sorted by most recent activity. Conversations can be filtered by status: Active, Waiting (guest hasn't replied), Resolved. Staff can assign a conversation to a specific team member. Staff can add internal notes (visible to staff only, never sent to guest).

**Key behaviours:**
- Conversation list: guest name/number, last message preview, timestamp, status badge, assignee avatar
- Status: Active (needs attention), Waiting (ball in guest's court), Resolved (done)
- Filter by status, assignee, date range
- Search conversations by guest name, phone number, or message content
- Assign conversation to team member → assignee gets notified
- Internal notes: yellow-tinted, clearly marked "internal," never sent via SMS
- Bulk resolve: select multiple conversations and mark resolved

**Tier:** All tiers (routing/assignment requires Professional+).

### F-008: Quick Actions

Pre-configured response templates for the most common requests. Staff taps a quick action, it populates the response field (editable before sending). Hotels can create custom quick actions.

**Key behaviours:**
- Default quick actions: restaurant recommendation, directions to hotel, checkout time, WiFi password, taxi booking confirmation
- Custom quick actions: hotel creates their own (title + template text with {guest_name} variable)
- Quick actions shown as chips above the response field
- Tapping a chip populates the field — does not auto-send
- Hotels can reorder and hide default quick actions

**Tier:** All tiers.

## Feature Area: Guest Intelligence

### F-009: Guest Profiles

Each phone number accumulates a guest profile over time. Profile shows: name (if provided), phone number, conversation history across visits, preferences extracted by AI, internal notes from staff.

**Key behaviours:**
- Profile created automatically on first message
- Staff can edit guest name and add notes
- AI auto-extracts preferences from conversations (dietary restrictions, room preferences, interests)
- Preferences shown as tags on the profile
- Conversation history grouped by visit (conversations within same check-in/check-out window)
- Profile accessible from conversation view (side panel)

**Tier:** Professional and Group.

### F-010: Guest Recognition

When a returning guest texts, the system matches their phone number to an existing profile and alerts staff: "Returning guest — stayed 3 times. Prefers high floor, allergic to shellfish." Staff sees context before responding.

**Key behaviours:**
- Automatic phone number matching
- Returning guest badge on conversation
- Previous preferences surfaced in conversation sidebar
- AI uses past preferences when drafting responses
- Cross-property recognition for Group tier (guest stayed at sister property)

**Tier:** Professional and Group (cross-property: Group only).

## Feature Area: Request Routing

### F-011: Request Categories & Routing

Incoming requests are auto-categorised by AI: Dining, Transport, Housekeeping, Activities, Information, Complaint, Other. Each category can be routed to a specific team member or role. Unrouted requests go to the general queue.

**Key behaviours:**
- AI categorises on arrival (shown as coloured tag)
- Routing rules: category → assigned team member
- Default: all categories route to general queue
- Staff can recategorise manually
- Routing only fires when no conversation assignee already exists

**Tier:** Professional and Group.

### F-012: Notifications

Staff receive notifications for: new conversation assigned to them, new message in an assigned conversation, conversation unattended for more than configured threshold (default: 5 minutes for active, 15 minutes for new). Notifications via browser push and optional email digest.

**Key behaviours:**
- Browser push notifications (opt-in)
- Email digest: daily summary of unresolved conversations (opt-in)
- Urgency escalation: if a conversation goes unattended past threshold, all admins are notified
- Notification preferences per staff member
- Sound on new message (toggleable)

**Tier:** All tiers.

## Feature Area: Hotel Configuration

### F-013: Property Settings

Hotel configures: property name, address, description, timezone, operating hours, logo, SMS greeting message (auto-sent on first contact), AI voice profile (casual/warm/formal), check-in and check-out times.

**Key behaviours:**
- All fields editable by admins
- SMS greeting sent once per guest, on their first message
- AI voice profile: 3 presets with example previews
- Check-in/check-out times used for visit grouping in guest profiles
- Changes take effect immediately

**Tier:** All tiers.

### F-014: Local Knowledge Base

Hotel builds a knowledge base of local information the AI uses to answer questions: restaurants (with cuisine type, price range, distance, reservation link), attractions, transport options, emergency contacts, hotel-specific FAQs.

**Key behaviours:**
- Add entries: title, category, description, optional link, optional hours
- Categories: Dining, Nightlife, Activities, Shopping, Transport, Services, Hotel FAQ
- AI references knowledge base entries when drafting responses
- Import: paste from a document or spreadsheet (AI-parsed)
- Entries can be marked seasonal (shown/hidden by date range)

**Tier:** All tiers (Starter limited to 50 entries, Professional+ unlimited).

### F-015: Custom AI Voice Training

Hotel provides 10-20 example Q&A pairs showing how they'd respond to common requests. The AI fine-tunes its tone to match. Also: a "brand voice" text description and list of words to use/avoid.

**Key behaviours:**
- Example Q&A pairs: staff writes the ideal response to 10-20 sample questions
- Brand voice description: free text, 1-2 paragraphs
- Words to use / words to avoid lists
- Preview: test the voice against sample questions before saving
- Voice training takes effect within 1 hour

**Tier:** Professional and Group.

## Feature Area: Analytics & Insights

### F-016: Dashboard Analytics

Overview screen showing: total conversations this period, AI-handled vs staff-handled ratio, average response time, busiest hours, most common request categories, guest satisfaction (based on conversation sentiment analysis).

**Key behaviours:**
- Period selector: today, 7 days, 30 days, custom range
- Key metrics as cards at the top
- Charts: conversations over time, category breakdown (pie), response time trend (line)
- AI-handled ratio shown as percentage with trend indicator
- Export as CSV

**Tier:** Professional and Group.

### F-017: Group Analytics (Multi-Property)

Roll-up view across all properties in a group. Compare properties by response time, volume, AI ratio, satisfaction. Identify best practices at top-performing properties.

**Key behaviours:**
- Property selector: all properties or individual
- Comparison table: properties ranked by key metrics
- Drill-down: click a property to see its individual dashboard

**Tier:** Group only.

## Feature Area: Billing & Subscription

### F-018: Subscription Management

Hotel selects a plan during onboarding (defaults to free trial → Starter). Admin can upgrade/downgrade from settings. Plan changes take effect at next billing cycle. Usage counters visible in settings.

**Key behaviours:**
- Stripe Checkout for initial subscription
- Stripe Customer Portal for plan changes, payment method updates, invoice history
- Usage counter: conversations this month / limit (Starter only)
- Approaching limit warning at 80% and 95%
- Overage: soft block — AI stops auto-responding, staff can still reply manually

**Tier:** All tiers.

## Explicitly Out of Scope for v1

- **PMS integrations** — no connection to Mews, Cloudbeds, Little Hotelier, or any property management system. Standalone only.
- **Voice calls** — SMS only. No voice AI, no phone tree.
- **Multi-language AI** — AI responds in English only. Staff can respond in any language manually.
- **Guest-initiated ratings** — no post-conversation survey or rating prompt.
- **Mobile app for staff** — dashboard is web-only (responsive). Native iOS/Android app is a v2 feature.
- **Automated booking** — AI can recommend a restaurant but cannot make a reservation through an integration (OpenTable, Resy). Staff handles bookings manually.
- **Marketing SMS** — no promotional messages, no broadcast, no drip campaigns.
- **White-label domain** — SMS is white-label, but the dashboard URL is always grove.app. Custom domains are v2.
- **Webhook/API for third parties** — no outbound webhooks or public API in v1 except Group tier's API access for PMS prep.
- **SSO/SAML** — email/password only.
