# Design System — Grove

## Design Direction

Grove should feel like the lobby of a well-designed boutique hotel — calm, warm, intentional. Every pixel should earn its place. The aesthetic draws from Aesop retail stores (natural materials, generous space, quiet confidence), The LINE hotel's brand identity (editorial without being cold), and Linear's information density (when you need to see a lot, you can, but the default state is spacious).

This is not a tech product. It is hospitality software. The difference: tech products optimize for productivity. Hospitality software optimizes for the feeling that everything is handled. The dashboard should make staff feel calm and in control, not busy and efficient.

**What to avoid:** Glassmorphism, gradients, neon accents, startup-bright colours, gamification patterns (streaks, badges, confetti), thick borders, heavy shadows. Nothing that screams "SaaS dashboard."

**What to pursue:** Natural colour palette, generous whitespace, quiet transitions, editorial typography, soft card edges, muted tones, information that breathes.

**Instrument Serif** for headings — a serif with quiet authority that feels like a boutique hotel's room number plate. Beautiful at large sizes, with details worth noticing. Google Fonts, free.

**Inter** for body/UI — the invisible workhorse. Legible at every size, warm enough not to feel clinical. Already the industry standard for a reason.

**Font loading:** Always `<link>` in `<head>`, never `@import`. Preconnect to Google Fonts. Fallback stack chosen for similar x-height and metrics.

<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

## Component Patterns

### Buttons

| Variant | Usage | Styling |
|---------|-------|---------|
| Primary | Main CTA | `--color-brand` bg, `--color-brand-text` text, `--radius-md`, `--shadow-xs` |
| Secondary | Secondary actions | Transparent bg, `--color-brand` text/border |
| Ghost | Tertiary actions | No bg, no border, `--color-text-secondary` text |
| Danger | Destructive actions | `--color-danger` bg, white text |

**States:** hover (slight lighten), focus (2px brand outline offset), disabled (40% opacity, no pointer), loading (spinner replacing text).

**Sizes:** sm (32px height, `--text-sm`), md (40px height, `--text-base`), lg (48px height, `--text-lg`).

### Message Bubbles

- **Guest (inbound):** Left-aligned, `--color-guest-bubble` bg, `--radius-lg` with bottom-left square corner.
- **Staff (outbound):** Right-aligned, `--color-staff-bubble` bg, `--radius-lg` with bottom-right square corner.
- **AI (outbound):** Right-aligned, `--color-ai-bubble` bg, small "AI" badge in corner.
- **Internal note:** Full-width, `--color-note-bg` bg, `--color-note-border` left border 3px, "Staff Only" label.

### Cards

`--color-surface` bg, `--shadow-xs` at rest, `--shadow-sm` on hover (if interactive). `--radius-lg`. `--space-5` padding. No border by default — shadow provides the edge.

### Form Inputs

40px height, `--color-surface` bg, `--color-border` border 1px, `--radius-md`. Focus: `--color-brand` border, subtle `--color-brand-muted` ring. Placeholder: `--color-text-tertiary`.

### Conversation List Item

64px height, full-width. Hover: `--color-surface-sunken` bg. Selected: `--color-brand-muted` bg with `--color-brand` left border 3px. Unread: `--weight-semibold` on name and preview.

### Sidebar Navigation

64px collapsed (icon only), 240px expanded. `--color-surface` bg. Items: 40px height, `--radius-md`, `--space-3` padding. Active item: `--color-brand-muted` bg, `--color-brand` text. Hover: `--color-surface-sunken` bg.

### Badges / Tags

`--radius-full`, `--text-xs`, `--space-1` vertical padding, `--space-2` horizontal. Category badges use semantic colours. Preference tags use `--color-accent-muted` bg, `--color-accent` text.

## Animation

**Principle:** Motion should feel like a deep breath, not a snap. Prefer easing-out (fast start, gentle stop). Never bounce. Never overshoot except on deliberate spring moments (like a notification badge appearing).

**Key animations:**
- **New message arriving:** Slide up from bottom of thread (200ms ease-out), subtle opacity fade. No bounce.
- **Conversation list reorder:** Item smoothly moves to top (300ms ease-out). Other items push down gently.
- **Sidebar expand/collapse:** Width transition (200ms ease-out). Labels fade in/out (120ms).
- **Page transitions:** Fade (150ms). No slide. No scale.
- **Notification badge:** Spring scale from 0 to 1 (400ms spring easing). This is the ONE place that has playful energy.
- **Toast messages:** Slide in from top-right (200ms ease-out), auto-dismiss after 5s with fade (300ms).
- **Modal/overlay:** Backdrop fade (200ms), content scale from 0.97 to 1 with fade (250ms ease-out).

**What does NOT animate:** Colour changes on hover (instant), text content changes, data loading (skeleton shimmer instead).

## Microcopy Tone

Grove's UI text should sound like a competent, warm colleague — not a robot, not a friend, not a corporate entity. The voice is the same one a good hotel GM uses with staff: clear, direct, respectful, occasionally dry.

| Context | Tone | Example |
|---------|------|---------|
| Toast (success) | Quiet confirmation | "Guest profile updated." |
| Toast (error) | Calm + actionable | "Message couldn't be sent. Check your connection and try again." |
| Empty state | Helpful, not cute | "No conversations yet. Share your Grove number with guests to get started." |
| Confirmation dialog | Direct, respectful | "Remove Sarah from your team? She'll lose access immediately." |
| Plan limit | Honest, not pushy | "You've used 450 of 500 conversations this month." |
| Button labels | Verb-first | "Send Message", "Add Entry", "Invite Team Member" |
| Error messages | Specific cause + fix | "That email is already registered. Try logging in instead." |
